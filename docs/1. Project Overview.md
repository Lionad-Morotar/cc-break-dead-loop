# cc-break-dead-loop 项目总览

## 项目是什么

`cc-break-dead-loop` 是一个 Claude Code 插件，自动检测并打断 agent 的两类死循环：

1. **主 agent Read 死循环**：主 agent 在 Read 工具返回 `"Wasted call — file unchanged since your last Read"` 或 `{ type: "file_unchanged" }` 时不理解信号，连续重复读取同一文件。
2. **子 agent 工具调用死循环**：子 agent（subagent / teammate）陷入连续重复调用同一工具（任意工具）。子 agent 无视 hook 的 `permissionDecision: 'deny'`（Claude Code 已知 bug #25000/#34692），常规 Hook 无法直接阻断。

## 核心目标

1. **双线检测**：线 1（主 agent Read，双 Hook）+ 线 2（子 agent 任意工具，watcher 常驻进程）
2. **渐进式干预**（线 1）：连续 3 次无效 Read 注入警告，5 次强制阻断
3. **绕过 subagent deny 失效**（线 2）：watcher 扫 subagent transcript 检测死循环，经 Stop hook `blockingError` 引导主 agent 调用 `TaskStopTool` 终止子 agent
4. **永不阻断正常操作**：多重错误边界确保插件 bug 不影响正常 Read / 不阻断 Claude Code 启动；watcher 崩溃自动重启
5. **隔离**：线 1 按 project + session + agent 隔离计数，线 2 按 sessionId 过滤告警

## 技术栈

| 维度 | 选择 |
|------|------|
| 语言 | JavaScript（ES Module，`.mjs`）|
| 运行时 | Node.js >= 18 |
| 测试框架 | Vitest ^4.1.8（devDependency），断言用 `node:assert` |
| 依赖 | **零运行时依赖**（Node.js 内置模块）；开发依赖 `vitest` |
| 构建 | **无构建步骤**，直接运行原生 JS |

## 核心特性

- **双线检测**：线 1（主 agent Read，双 Hook）+ 线 2（子 agent 工具死循环，watcher）
- **watcher 常驻进程**：detached spawn，心跳保活（30s 超时重启），崩溃自愈
- **绕过 subagent deny 失效**：子 agent 死循环由主 agent 经 `TaskStopTool` 终止（Stop hook `blockingError` + `exit 2`）
- **多重检测兼容**（线 1）：字符串 `"Wasted call"` / `{ type: "file_unchanged" }` 对象 / `content` 对象 / `JSON.stringify` 兜底
- **参数精确比较**（线 1）：`===` 直接比较，不规范化 `undefined` → `0`
- **稳定序列化指纹**（线 2）：对象键排序后 stringify，键序不同的等价参数视为相同
- **原子写入**：`state.json`、`alerts.json` 均 `writeFile(tmp) → rename(dest)`
- **多重错误边界**：handler try/catch + node-runner graceful fallback + watcher 崩溃自愈 + Setup 永不阻断

## 安装

### 方式一：Marketplace 安装（推荐）

```bash
# 在 Claude Code CLI 中
/plugin marketplace add Lionad-Morotar/cc-break-dead-loop
/plugin install cc-break-dead-loop
```

### 方式二：NPX CLI 安装

```bash
npx cc-break-dead-loop install
```

该命令会自动：
1. 复制 `plugin/` 到 `~/.claude/plugins/marketplaces/<owner>/plugin/`
2. 注册到 `known_marketplaces.json`
3. 注册到 `installed_plugins.json`
4. 启用插件（写入 `settings.json`）

### 验证安装

重启 Claude Code，启动日志中应出现：

```
[cc-break-dead-loop] Setup: OK (Node.js v22.22.1)
[cc-break-dead-loop] Watcher start (pid=12345)
```

### 其他 CLI 命令

```bash
npx cc-break-dead-loop status            # 查看安装状态
npx cc-break-dead-loop uninstall         # 卸载插件
npx cc-break-dead-loop uninstall --purge # 卸载并删除 marketplace 目录
```

## 已知限制

- **线 2 依赖主 agent 遵循引导**：watcher 检测到子 agent 死循环后，通过 Stop hook `blockingError`（`exit 2`）+ PostToolUse `additionalContext` 引导主 agent 调用 `TaskStopTool`。若主 agent 忽略 `blockingError`，仍可能漏阻断。
- **watcher 检测有延迟**：扫描间隔 5s + transcript 落盘延迟，子 agent 死循环最快在数秒后被发现。
- **subagent/teammate 无视 `permissionDecision: 'deny'`**（Claude Code bugs #25000, #34692）：线 1 的 deny 对主 agent 完全有效，对子 agent 无效（故引入线 2 watcher 机制）。
- **独立 agent_id → 独立计数**（线 1）：不同 agent 各自维护计数器，互不影响。
