# cc-break-dead-loop 项目总览

## 项目是什么

`cc-break-dead-loop` 是一个 Claude Code 插件，自动检测并打断 agent 对同一未改动文件的连续 Read 死循环。当子代理（如 gsd-planner）反复读取同一个文件，且 Claude Code 每次都返回 "Wasted call — file unchanged since your last Read" 时，本插件会介入阻断这种无意义的循环。

## 核心目标

1. **检测死循环**：通过 PostToolUse Hook 监听 Read 工具的 "Wasted call" / `file_unchanged` 信号
2. **渐进式干预**：连续 3 次无效读取时注入警告，5 次时强制阻断（`permissionDecision: 'deny'`）
3. **永不阻断正常操作**：双重错误边界确保插件自身 bug 不会影响正常的 Read 工具使用
4. **多代理隔离**：按 project + session + agent 隔离检测状态，互不干扰

## 技术栈

| 维度 | 选择 |
|------|------|
| 语言 | JavaScript（ES Module） |
| 运行时 | Node.js >= 18 |
| 测试框架 | `node:test`（Node.js 内置） |
| 依赖 | **零外部依赖**，仅使用 Node.js 内置模块 |
| 构建 | **无构建步骤**，直接运行原生 JS |

## 核心特性

- **双 Hook 机制**：PostToolUse（检测）+ PreToolUse:Read（拦截）协同工作
- **状态持久化**：文件系统原子写入，避免并发损坏
- **参数精确比较**：`===` 直接比较，不规范化 `undefined` → `0`
- **三层工具响应检测**：字符串 / `file_unchanged` 对象 / `content` 对象 / `JSON.stringify` 兜底（D6）
- **双重阻断保险**：`permissionDecision: 'deny'`（主 agent）+ `additionalContext`（subagent/teammate）
- **Graceful Fallback**：任何内部错误都静默降级，不阻断正常 Read
- **Setup 环境检测**：插件加载时自动检测 Node.js 版本

## 快速开始

```bash
# 运行全部测试
npm test

# 运行单个测试文件
node --test tests/state.test.mjs

# 手动模拟 Hook 输入
echo '{"tool_name":"Read","tool_input":{"file_path":"/a/b"},"tool_response":{"type":"file_unchanged","file":{"filePath":"/a/b"}},"session_id":"s","agent_id":"a","cwd":"/tmp"}' | node plugin/src/index.mjs post-tool-use
```

## 安装

### 方式一：Marketplace 安装（推荐）

在 Claude Code 中直接添加 marketplace：

```bash
# 在 Claude Code CLI 中
/plugins add lionad-morotar
```

### 方式二：NPX CLI 安装

```bash
npx cc-break-dead-loop install
```

该命令会自动：
1. 复制 `plugin/` 到 `~/.claude/marketplace/lionad-morotar/plugin/`
2. 注册到 `known_marketplaces.json`
3. 注册到 `installed_plugins.json`
4. 启用插件（写入 `settings.json`）

### 方式三：手动安装

将 `plugin/` 目录复制到 Claude Code plugins 目录：

```bash
mkdir -p ~/.claude/plugins/cc-break-dead-loop
cp -r plugin/* ~/.claude/plugins/cc-break-dead-loop/
```

### 验证安装

重启 Claude Code，启动日志中应出现：

```
[cc-break-dead-loop] Setup: OK (Node.js v22.22.1)
```

### 其他 CLI 命令

```bash
npx cc-break-dead-loop status       # 查看安装状态
npx cc-break-dead-loop uninstall    # 卸载插件
npx cc-break-dead-loop uninstall --purge  # 卸载并删除 marketplace 目录
```

## 已知限制

- **Subagent/teammate 无视 `permissionDecision: 'deny'`**（Claude Code bugs #25000, #34692）：deny 对主 agent 完全有效，但 subagent/teammate 类型 agent 可能无视阻断。插件通过同时注入 `additionalContext` 作为双重保险，引导 subagent 主动停止。
- **独立 agent_id → 独立状态计数**：不同 agent 各自维护计数器，互不影响。
- **主 agent 场景完全有效**：`permissionDecision: 'deny'` 对主 agent 强制阻断，无已知问题。
