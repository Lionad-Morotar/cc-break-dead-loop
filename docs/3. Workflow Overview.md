# 工作流总览

## 核心工作流

### 工作流 1：插件加载与初始化

```mermaid
sequenceDiagram
  participant CC as Claude Code
  participant SC as setup-check.mjs
  participant Node as Node.js

  CC->>SC: Setup Hook 触发（spawn 子进程）
  SC->>Node: spawnSync('node', ['--version'])
  Node-->>SC: v22.22.1
  SC->>Node: spawnSync('git', ['--version'])
  Node-->>SC: git version 2.47.0
  SC->>CC: stdout: "[cc-break-dead-loop] Setup: OK"
  SC->>CC: exit(0) ← 永不阻断启动
```

**步骤**：
1. Claude Code 启动时加载插件，触发 Setup Hook
2. `setup-check.mjs` 检测 Node.js 是否在 PATH 且版本 >= 18
3. 可选检测 Git 是否在 PATH（用于仓库名解析）
4. 全部通过则 stdout 输出 "OK"，失败则 stderr 输出警告
5. 无论结果如何，exit(0)，永不阻断 Claude Code 启动

### 工作流 2：PostToolUse 检测（Read 执行后）

```mermaid
sequenceDiagram
  participant CC as Claude Code
  participant NR as node-runner.mjs
  participant IDX as index.mjs
  participant HND as handlers.mjs
  participant ST as state.mjs
  participant FS as 文件系统

  CC->>NR: PostToolUse Hook（stdin: Read 结果 JSON）
  NR->>NR: 收集 stdin（5s 超时保护）
  NR->>IDX: main('post-tool-use', stdinData)
  IDX->>IDX: JSON.parse(stdinData)
  IDX->>HND: postToolUse(input)
  HND->>HND: toolName === 'Read' ?
  alt 不是 Read
    HND-->>IDX: { continue: true, suppressOutput: true }
  else 是 Read
    HND->>HND: isWastedCall(toolResponse)
    alt 非 wasted call
      HND-->>IDX: { continue: true, suppressOutput: true }
    else 是 wasted call
      HND->>ST: incrementCounter(stateDir, params)
      ST->>FS: readState(stateDir)
      FS-->>ST: state.json 或 null
      ST->>ST: isSameReadParams(state, filePath, offset, limit)
      alt 参数相同
        ST->>ST: consecutiveWastedReads++
      else 参数不同
        ST->>ST: 重置为 1，更新参数
      end
      ST->>FS: writeState(stateDir, newState)（原子写入）
      ST-->>HND: 返回新计数
      HND-->>IDX: { continue: true, suppressOutput: true }
    end
  end
  IDX-->>NR: result
  NR-->>CC: stdout: JSON.stringify(result) + exit(0)
```

**步骤**：
1. Claude Code 执行 Read 工具后，触发 PostToolUse Hook
2. `node-runner.mjs` 收集 stdin，调用 `main('post-tool-use', data)`
3. `index.mjs` 解析 JSON 并分发到 `postToolUse()`
4. `handlers.mjs` 检查 toolName 是否为 "Read"，多模式检测 "Wasted call"
5. 若检测到 wasted call，调用 `incrementCounter()` 更新状态
6. 状态管理模块比较当前 Read 参数与状态记录，相同则递增，不同则重置
7. 原子写入更新后的状态文件
8. 返回 `{ continue: true, suppressOutput: true }`

### 工作流 3：PreToolUse:Read 拦截（Read 执行前）

```mermaid
sequenceDiagram
  participant CC as Claude Code
  participant NR as node-runner.mjs
  participant IDX as index.mjs
  participant HND as handlers.mjs
  participant ST as state.mjs
  participant FS as 文件系统

  CC->>NR: PreToolUse Hook（stdin: 即将执行的 Read 参数）
  NR->>NR: 收集 stdin
  NR->>IDX: main('pre-tool-use-read', stdinData)
  IDX->>HND: preToolUseRead(input)
  HND->>ST: readState(stateDir)
  ST->>FS: 读取 state.json
  FS-->>ST: state 或 null
  HND->>HND: isSameReadParams(state, currentParams)

  alt 状态不存在 或 参数不同
    HND-->>IDX: { continue: true, suppressOutput: true } ← 放行
  else 参数匹配
    HND->>HND: count = state.consecutiveWastedReads
    alt count < 3
      HND-->>IDX: { continue: true, suppressOutput: true } ← 放行
    else 3 <= count < 5
      HND-->>IDX: { continue: true, suppressOutput: false, hookSpecificOutput: { additionalContext: "⚠️ 警告..." } }
    else count >= 5
      HND-->>IDX: { shouldBlock: true, systemMessage: "检测到 Read 死循环..." }
    end
  end

  IDX-->>NR: result
  alt result.shouldBlock
    NR-->>CC: stdout: { systemMessage } + exit(2) ← 阻断
  else 正常
    NR-->>CC: stdout: JSON.stringify(result) + exit(0)
  end
```

**步骤**：
1. Claude Code 准备执行 Read 工具前，触发 PreToolUse Hook
2. `preToolUseRead()` 读取当前状态，比较 Read 参数
3. 若参数不匹配或状态不存在，直接放行（`continue: true`）
4. 若参数匹配，根据计数器值决策：
   - `< 3`：放行
   - `>= 3 且 < 5`：注入 `additionalContext` 警告，但允许执行
   - `>= 5`：返回阻断标记，`node-runner.mjs` 输出 `systemMessage` 并 exit(2)

## 状态管理

### 状态隔离维度

状态文件按三层目录结构隔离：

```
~/.data/cc-break-dead-loop/
  └─ <safe-project-name>/          ← Git 仓库名 或 cwd 文件夹名
      └─ <session-id>/              ← Claude Code 会话 ID
          └─ <safe-agent-name>/     ← agent_id（空值为 "main"）
              └─ state.json
```

### 状态文件结构

```json
{
  "sessionId": "sess-abc-123",
  "filePath": "/Users/lionad/project/src/main.ts",
  "offset": 10,
  "limit": 50,
  "consecutiveWastedReads": 4,
  "lastUpdatedAt": "2026-05-10T05:30:00.000Z"
}
```

### 计数器重置规则

当 Read 参数任一变化时，计数器重置为 1：
- `file_path` 变化（读取不同文件）
- `offset` 变化（读取不同位置）
- `limit` 变化（读取不同行数）

**注意**：`offset: undefined` 和 `offset: 0` 被视为不同参数（D7 决策），不会相互重置。

## 错误处理

### 分级错误边界

| 层级 | 处理者 | 行为 | 场景 |
|------|--------|------|------|
| L1 | node-runner.mjs | 异常 → `{ continue: true }` + exit(0) | runner 自身崩溃 |
| L2 | index.mjs | JSON 解析失败 → `{ continue: true }` | stdin 非有效 JSON |
| L3 | index.mjs | handler 抛出 → `{ continue: true }` | handler 内部 bug |
| L4 | handlers.mjs | 参数缺失 → 静默跳过 | tool_input 缺少 file_path |
| L5 | state.mjs | 读取失败 → 返回 null | 状态文件不存在或损坏 |

### 阻断 vs 放行

```mermaid
flowchart TD
  A[插件内部错误] --> B[静默放行<br/>continue: true]
  C[状态文件不存在] --> B
  D[Read 参数变化] --> B
  E[非 Read 工具] --> B
  F[count < 3] --> B
  G[count 3-4] --> H[注入警告<br/>additionalContext]
  I[count >= 5] --> J[强制阻断<br/>exit(2)]
```
